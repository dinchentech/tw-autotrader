## 🤔 什麼是回溯？

**回溯（Backtest）** 又稱 **回測**，是量化交易的核心步驟。它的概念非常簡單：

> **用過去的歷史數據，去驗證你的交易策略如果在那個當下執行，到底會不會賺錢。**

舉例來說：如果你發明了一套號稱能穩定獲利的策略，你不能直接拿真錢去市場上測試。你應該先讓程式讀取過去三年（2023~2026 年）台積電（2330）的每日股價資料，然後模擬在過去的每一天，策略是否會觸發買進或賣出訊號，最後統計這段期間的總報酬、勝率等績效指標。

### 回溯的核心價值

| 目的 | 說明 |
|------|------|
| **驗證策略有效性** | 在投入真實資金之前，先用歷史數據檢驗策略是否真的能賺錢 |
| **參數最佳化** | 調整策略參數（如均線週期、標準差倍數），找出歷史表現最好的組合 |
| **了解策略特性** | 了解策略在什麼市場環境（震盪盤、大牛市、崩盤）表現好或差 |
| **建立交易信心** | 經過足夠的回溯驗證後，在實盤執行時才不會因為短線虧損而恐慌 |

### 回溯的局限性

> ⚠️ **過去績效不代表未來獲利。** 回溯結果僅供參考，實盤交易會受到：
> - **滑價（Slippage）**：實際成交價格可能與訊號觸發價格不同
> - **手續費與稅金**：交易成本會侵蝕獲利
> - **流動性不足**：零股交易可能無法以理想價格成交
> - **未來函數（Look-ahead Bias）**：策略設計時不小心使用了「當下還不知道的未來資料」

## 🔄 三種回溯模式

### 模式一：`backtest.py` — Yahoo Finance 批次回溯

這是**主要回溯工具**，使用 Yahoo Finance 的免費歷史股價資料。

**特性：**
- ✅ 資料源：Yahoo Finance（無需 API Key，完全免費）
- ✅ 自動掃描 `config/symbols.py` 中定義的所有標的（目前 8 檔：2330、2454、2317、2881、0050、0056、00632R、00646）
- ✅ 單一策略執行，一次跑完全部標的
- ✅ 自動匯出 CSV 績效報表
- ✅ 支援命令列參數：`--strategy` 選擇策略、`--start` 設定開始日期
- ❌ 不支援複委託手續費與證交稅計算

**適用場景：** 快速驗證單一策略在整個觀察清單上的表現。

### 模式二：`backtest_finmind.py` — FinMind 多策略比較

**進階回溯工具**，使用 FinMind 台灣股市資料庫。

**特性：**
- ✅ 資料源：FinMind（台灣在地資料庫，需免費註冊取得 API Token）
- ✅ 四種策略同時跑在同一支股票上，方便橫向比較
- ✅ 支援手續費設定：買進 0.1425%、賣出 0.4425%（含證交稅）
- ✅ 回傳完整績效數據：最終權益、交易次數、勝率、平均報酬

**適用場景：** 針對單一標的，比較四種策略哪個歷史表現最好。

### 模式三：`backtest_esun.py` — 玉山 E.Sun SDK 回溯

使用玉山證券提供的 SDK 擷取歷史行情資料進行回測。

**特性：**
- ✅ 資料源：玉山 E.Sun MarketData API（需先完成 SDK 安裝）
- ✅ 支援 `--symbol` 指定單一標的
- ✅ 支援 `--strategy` 指定策略（vwap / ma_cross / bollinger / breakout）
- ✅ 使用 `config.simulation.ini.example` 中的 API 設定
- ✅ 支援函式版策略（`strategies/vwap_deviation.py` 等）
- 🚫 **限單一標的**，非批次執行

**適用場景：** 使用券商的真實歷史行情資料驗證策略，避免 Yahoo Finance / FinMind 的資料偏差。

### 模式四：`simulate_portfolio.py` — 投資組合模擬

將回測結果延伸為完整投資組合模擬，支援**兩種資金模型**：

| 模型 | 說明 | 適用情境 |
|------|------|---------|
| **DCA（每月定額）** | 每月投入固定金額，依權重分配至各策略 | 每月 NT$20,000 小資定期定額 |
| **一筆資金（Lump Sum）** | 初始資金一次到位，獲利可再投入 | NT$500,000 一筆資金配置 |

**輸出：** 自動產生 Markdown 回溯報告（含各標的明細、總績效、策略比較表）

## 🎯 支援的四種策略

### 1. VWAP 偏離度反轉（`vwap`）

> **核心邏輯：** 當股價遠低於法人平均成本（VWAP）且市場極度恐慌（RSI < 30）時抄底；當股價遠高於法人成本且過熱時賣出。

| 參數 | 預設值 | 單位 | 說明 |
|------|--------|------|------|
| `sigma_mult` | `1.5` | 倍數 | VWAP 偏離倍數（越大越難觸發） |
| `rsi_period` | `5` | 根 K 線 | RSI 計算週期（越短越敏感，5 ≈ 一週） |

**適合標的：** 金融股（富邦金 2881、國泰金 2882、兆豐金 2886）

### 2. 均線交叉（`ma_cross`）

> **核心邏輯：** 快線（MA9）上穿慢線（MA21）→ 黃金交叉買進；快線下穿慢線 → 死亡交叉賣出。純粹的順勢策略。

| 參數 | 預設值 | 單位 | 說明 |
|------|--------|------|------|
| `fast_period` | `9` | 根 K 線 | 快速均線週期（9 ≈ 一週半） |
| `slow_period` | `21` | 根 K 線 | 慢速均線週期（21 ≈ 一個月） |

**適合標的：** 半導體權值股（台積電 2330、聯發科 2454）
**⚠️ 注意：** 盤整期容易被假訊號雙巴。

### 3. 布林通道反轉（`bollinger`）

> **核心邏輯：** 價格跌破下軌 + RSI < 30 → 超跌買進；價格漲破上軌 + RSI > 70 → 超買賣出。逆勢抄底策略。

| 參數 | 預設值 | 單位 | 說明 |
|------|--------|------|------|
| `window` | `20` | 根 K 線 | 布林通道計算週期（20 ≈ 一個月） |
| `std_dev` | `2.0` | 倍數 | 標準差倍數（越大通道越寬，2 = 95.4% 信賴區間） |
| `rsi_period` | `5` | 根 K 線 | RSI 計算週期（越短越敏感） |

**適合標的：** 市值型與高股息 ETF（0050、006208、00878、0056）

### 4. 突破交易（`breakout`）

> **核心邏輯：** 價格突破 N 日新高 + ATR 波動度足夠大 → 猜測主力進場，順勢買進；跌破 N 日新低 → 主力撤退賣出。

| 參數 | 預設值 | 單位 | 說明 |
|------|--------|------|------|
| `lookback` | `25` | 根 K 線 | 突破回溯期間（2025-06 前為 20，調整為 25 降低假突破） |
| `atr_period` | `14` | 根 K 線 | ATR 計算週期（14 ≈ 三週，值越大波動參考越平滑） |
| `atr_threshold` | `0.02` | 比率 | ATR/股價門檻（2025-06 前為 0.01，調至 0.02 過濾更多盤整訊號） |

**適合標的：** AI 供應鏈高價股（廣達 2382、緯穎 6669、世芯-KY 3661）

## 🚀 如何執行回溯

### 環境準備

```bash
# 安裝依賴套件
pip install -r requirements.txt
pip install python-dotenv pandas numpy
```

### 執行 Yahoo Finance 回溯

```bash
# 使用預設策略（vwap），從 2023-01-01 開始
python backtest.py

# 指定策略
python backtest.py --strategy ma_cross

# 指定策略與開始日期
python backtest.py --strategy bollinger --start 2022-06-01

# 調整策略參數（免改原始碼）
python backtest.py --strategy ma_cross --fast_period 5 --slow_period 30
python backtest.py --strategy bollinger --std_dev 2.5 --rsi_period 7
python backtest.py --strategy breakout --lookback 40
python backtest.py --strategy vwap --sigma_mult 2.0

# 查看所有可用參數
python backtest.py --help

# 可用的策略名稱
python backtest.py --strategy vwap      # VWAP 偏離度
python backtest.py --strategy ma_cross  # 均線交叉
python backtest.py --strategy bollinger # 布林通道反轉
python backtest.py --strategy breakout  # 突破交易
```

### 執行 FinMind 回溯

```bash
# 需先設定 FinMind API Token
# 1. 前往 https://finmindtrade.com/ 免費註冊
# 2. 在 .env 檔案中加入：
#    FINMIND_API_TOKEN=your_token_here

# 執行 FinMind 回溯（預設 2330，四策略同時比較）
python backtest_finmind.py
```

### 執行玉山 E.Sun 回溯

```bash
# 需先安裝 E.Sun SDK（.whl 檔在 esun_sdk/ 目錄下）
pip install esun_sdk/esun_marketdata-*.whl esun_sdk/esun_trade-*.whl

# 單一標的回溯（需設定 .env 中的 ESUN_* 參數）
python backtest_esun.py --symbol 2330 --strategy vwap

# 指定回測期間
python backtest_esun.py --symbol 2881 --strategy vwap --start 2024-01-01 --end 2025-12-31
```

### 執行投資組合模擬

```bash
# 每月 NT$20,000 DCA 模擬（4 檔標的）
python simulate_portfolio.py --mode dca --monthly 20000

# NT$500,000 一筆資金模擬（8 檔標的）
python simulate_portfolio.py --mode lumpsum --capital 500000
```

### 指定回溯標的

`backtest.py` 預設會掃描 `config/symbols.py` 中的所有標的：


## 📊 績效指標說明

執行回溯後，系統會為每檔股票計算以下績效指標：

### 單一股票績效

```
  → 回測 2330 (2330.TW)...
    ✅ 交易次數: 24, 勝率: 62.5%, 總報酬: 18.32%
```

| 指標 | 計算方式 | 說明 |
|------|----------|------|
| **交易次數** | 策略觸發的買進 + 賣出訊號總數 | 次數太少可能樣本不足；次數太多可能過度交易 |
| **勝率** | 獲利交易次數 ÷ 總交易次數 | 60% 以上通常算不錯 |
| **總報酬** | 所有交易報酬的加總 | 正值代表策略在回測期間整體賺錢 |
| **平均每筆報酬** | 總報酬 ÷ 交易次數 | 用來評估每筆交易的平均品質 |

### 報酬計算邏輯

```python
# 買進訊號（做多）：下一根 K 的漲跌幅
報酬 = (下一根收盤價 - 這根收盤價) / 這根收盤價

# 賣出訊號（做空）：下一根 K 的跌跌幅
報酬 = (這根收盤價 - 下一根收盤價) / 這根收盤價
```

### 回測總結

當所有標的都跑完後，系統會輸出總結：

```
📈 回測總結
============================================================
2330   | 交易: 24 次 | 勝率: 62.5% | 報酬: 18.32%
2454   | 交易: 18 次 | 勝率: 55.6% | 報酬: 12.45%
...
------------------------------------------------------------
平均   | 交易: 42 次 | 勝率: 58.2% | 報酬: 14.88%
```

**總結中的「平均」是將所有標的的勝率與報酬簡單平均，並非資金加權計算。**

## 🔬 回溯流程解析

以下是 `backtest.py` 的完整執行流程，幫助您深入了解系統如何運作：

```
步驟 1：讀取策略與參數
  └─ 從命令列或環境變數取得策略名稱（預設 vwap）
  └─ 從 STRATEGY_CONFIG 映射表取得對應的策略函式與參數

步驟 2：初始化策略引擎
  └─ StrategyEngine(strategy_func, **params)
  └─ 引擎本身非常輕量，主要負責轉發呼叫

步驟 3：對每個標的依序執行
  └─ 遍歷 ALL_SYMBOLS（目前 8 檔）
  └─ 根據股票代號決定 Yahoo Finance 後綴（.TW / .TWO）
  └─ 呼叫 load_historical_data() 下載歷史股價
  └─ 若資料為空則跳過該標的

步驟 4：執行策略
  └─ engine.run(df) → 呼叫策略函式
  └─ 策略函式在 DataFrame 中新增 signal 欄位
  └─ signal = 1（買進）, -1（賣出）, 0（無動作）

步驟 5：計算績效
  └─ calculate_performance(df)
  └─ 根據 signal 欄位計算每筆交易的報酬
  └─ 回傳交易次數、勝率、總報酬

步驟 6：匯出結果
  └─ 將所有非空的結果寫入 CSV
  └─ 檔案命名包含策略名稱與時間戳
```

### Signal 產生邏輯圖解

```
DataFrame（範例）：
┌──────────┬────────┬────────┐
│  日期    │ close  │ signal │
├──────────┼────────┼────────┤
│ 2023-01-03│ 450.00 │   0    │ ← 無訊號
│ 2023-01-04│ 445.50 │   0    │
│ 2023-01-05│ 439.00 │   1    │ ← 買進訊號：跌破下軌 + RSI < 30
```

## ❓ 常見問題

### Q1：為何有些標的會顯示「資料為空，跳過」？

Yahoo Finance 對某些 ETF（如 00632R 反向 ETF）或冷門股的歷史資料可能不完整。建議：
- 改用 FinMind 資料源（`backtest_finmind.py`）
- 檢查 `config/symbols.py` 中的股票代號是否正確

### Q2：為何出現「無有效交易訊號」？

可能原因：
- **回測期間太短**：部分策略（如均線交叉）需要較長時間才能產生訊號
- **參數太嚴格**：例如 `sigma_mult` 設定太高，導致股價從未偏離到觸發條件
- **策略不適合該標的**：逆向策略在強勢趨勢股上可能很少觸發

建議：延長回測期間（如從 2020 年開始），或放寬策略參數進行測試。

### Q3：回溯結果和實盤績效為何有落差？

這是所有量化交易系統都無法避免的問題，主要原因：
- **沒有計算交易成本**：目前回溯不包含手續費（0.1425%）、證交稅（賣出 0.3%）
- **沒有考慮滑價**：假設以收盤價成交，但實盤可能買貴或賣便宜
- **零股流動性**：高價股的零股交易量不足，可能無法順利成交

### Q4：如何調整回溯的股票清單？

直接編輯 [`config/symbols.py`](file:///mnt/c/Users/frank/Documents/github/tw-autotrader/config/symbols.py)：

### Q5：backtest.py 和 backtest_finmind.py 有什麼不同？

| 比較項目 | `backtest.py` | `backtest_finmind.py` |
|----------|---------------|----------------------|
| 資料源 | Yahoo Finance（免費） | FinMind（需註冊） |
| 支援標的 | 多標的批次執行 | 單一標的 |
| 策略數量 | 一次一種 | 四種同時比較 |
| 手續費計算 | ❌ 無 | ✅ 有（可設定） |
| 適合場景 | 快速掃描觀察清單 | 深度分析特定股票 |

### Q6：backtest.py 跑完，warm-up 資料會自動餵給 live_trader 嗎？

**不會。** `backtest.py` 和 `live_trader*.py` 是完全獨立的兩條管道：

```
backtest.py (Yahoo Finance)              live_trader_finmind.py (券商 API)
        │                                         │
        │ 下載歷史資料                             │ 啟動時自動抓取 30 天歷史資料
        │ 跑策略 → 算績效 → 匯出 CSV              │ 跑策略 → 監控訊號 → 下單
        │                                         │
        └── 結束後不存任何暖身資料 ──→              └── 內建 warm-up，不需額外餵資料
```

**它們之間傳遞的是「參數」，不是「資料」：**

```bash
# 1. 回測找出最佳參數
python backtest.py --strategy ma_cross --fast_period 5 --slow_period 30

# 2. 把參數寫入 .env
# MA_CROSS_FAST_PERIOD=5
# MA_CROSS_SLOW_PERIOD=30

# 3. live_trader 啟動時自己抓 30 天歷史資料 + 用同一組參數
python live_trader_finmind.py --symbol 2330 --strategy ma_cross
```

| 問題 | 答案 |
|------|------|
| `backtest.py` 跑完會存 warm-up 資料給 live trader 嗎？ | ❌ 不會，只產出 CSV 績效報表 |
| live trader 自己會抓 warm-up 資料嗎？ | ✅ 會，啟動時自動抓取 30 天歷史資料（約 22 根交易日 K 線），足夠所有策略暖身 |
| 那 warm-up 的提醒是給誰的？ | **給回測用的。** 回測時 `--start` 要設夠早（至少 1 年），否則前 22 根 K 線都是 junk，回測結果不準。實盤不用擔心暖身問題。 |

## 💡 實戰建議

### 回測 SOP（建議流程）

```
第 1 步：先用預設參數跑所有策略
  python backtest.py --strategy vwap
  python backtest.py --strategy ma_cross
  python backtest.py --strategy bollinger
  python backtest.py --strategy breakout

第 2 步：找出表現最好的策略與標的組合

第 3 步：針對該組合調整參數
  python backtest.py --strategy bollinger --start 2020-01-01 --std_dev 2.5 --rsi_period 7
  （直接用命令列調整參數，不需修改任何原始碼）

第 4 步：至少跑 3 年以上的資料
  確保策略經歷過不同市場環境（多頭、空頭、盤整）

第 5 步：確認策略邏輯在實盤合理
  檢查每個訊號觸發的日期，確認沒有未來函數

第 6 步：用模擬交易測試 1~2 週
  先用 live_trader_finmind.py 的模擬模式觀察
```

### 參數調整方向

| 目標 | 調整方式 |
|------|----------|
| 增加交易次數 | 放寬觸發條件（降低 `sigma_mult`、`std_dev`） |
| 減少假訊號 | 提高觸發門檻（提高 RSI 閾值、加入成交量過濾） |
| 提高勝率 | 增加過濾條件（如大盤趨勢判斷） |
| 抓住大波段 | 延長突破回溯期（`lookback` 從 20 改為 40） |

---

> 📌 **請記住：回測的目的是「排除壞策略」，而不是「找出完美策略」。**  
> 一個在各種市場環境下都能穩定小賺的策略，遠比一個在大牛市暴賺但在崩盤時慘賠的策略更有價值。
>
> **先用回測驗證，再用小額資金實盤測試，最後才放大資金。** 🛡️
