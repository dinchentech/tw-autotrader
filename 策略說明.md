## 前導概念：三大核心邏輯

以下為 AI 對台股自動化交易三大核心邏輯的白話說明：

### 1. 成交量爆發（Volume Spike）
散戶無法在同一天同時買進數萬張股票。當成交量突然放大到過去 5 天或 20 天平均值的 2 到 3 倍以上時，程式就會推判定「這絕對不是散戶按出來的」，而是大盤法人或主力資金在大舉進駐。策略對應：專案中的量價篩選器，會去計算 Volume > MA(Volume, 20) * N。

### 2. 價格突破區間（Price Breakout）
當大資金（如外資、投信）為了建立幾千張的部位時，會連續不斷地吃單，這會導致原本橫盤整理的股價強勢突破過去 5 日或 20 日的最高價。策略對應：這類似於經典的「唐奇安通道（Donchian Channel）」或「海龜交易法則（Turtle Trading）」。當今日收盤價大於過去 N 天的最高價，策略就會「猜測」主力已經進場拉抬，隨即觸發買進訊號順勢上車。

### 3. 法人籌碼外掛
直接看證據。當「投信連續買超 3 天以上」或「外資與投信聯手同步買進」時。在台股，投信（本土基金）的進出常常代表一波中線波段的起漲點。

## 1. 布林通道反轉 + RSI 過濾（Bollinger）

**一句話**：股價跌到統計學極限 + 市場極度恐慌時買進；漲到統計學極限 + 市場極度貪婪時賣出。

📺 **動畫演示**：[點擊觀看](https://dinchentech.github.io/tw-autotrader/bollinger-animation.html) — 以走勢動畫解說買賣點邏輯

### 策略原理

布林通道（Bollinger Bands）以 20 日移動平均線為中軸，上下各 2 倍標準差構成通道。根據統計學常態分佈，約 95.4% 的價格會落在通道內；衝出通道即代表市場進入極端狀態。結合 RSI（5 日）判讀市場情緒，形成雙重過濾。

數學定義：

| 元件 | 公式 | 意義 |
|------|------|------|
| BB_Middle | SMA(close, 20) | 平均成本（價值中軸） |
| BB_Std | STD(close, 20) | 近期波動劇烈程度 |
| BB_Upper | Middle + 2 × Std | 統計學天花板（超買門檻） |
| BB_Lower | Middle − 2 × Std | 統計學地板（超賣門檻） |
| RSI(5) | 100 − 100 / (1 + RS) | 極短線多空力道 |

**AI 白話解說**：布林通道是由約翰·布林格（John Bollinger）在 1980 年代發明的技術指標，其核心原理是基於統計學上的「均值回歸（Mean Reversion）」。程式碼中使用 `std_dev=2.0`（2 倍標準差），統計學上有 95.4% 的 K 線收盤價會乖乖待在上下軌之內。一旦股價衝出這兩條線，就代表市場進入了極端狀態。而 `rsi_period=5` 是一個非常敏銳的短週期設定，能快速捕捉過去 5 天內買方與賣方的力道。

**買進訊號（抓黃金冰點）**：價格跌破下軌 ＋ RSI < 30。布林下軌代表股價已經跌到了過去 20 天 95.4% 常態分佈的最底層；RSI < 30 代表市場充滿恐慌，散戶不計代價拋售。當兩個條件同時成立，代表這檔股票在統計上已經超跌，且散戶恐慌到了極點，此時通常會吸引搶反彈的資金或主力進場「抄底」，股價在幾天內強勢反彈（均值回歸）的機率極高。

**賣出訊號（抓瘋狂頂點）**：價格漲破上軌 ＋ RSI > 70。布林上軌代表股價衝到了統計學上的天花板；RSI > 70 代表市場情緒極度亢奮貪婪。此時短線獲利盤極其豐厚，隨時會有沉重的賣壓湧出，程式果斷賣出能將獲利鎖在相對最高點。

### 買進條件

| 條件 | 說明 |
|------|------|
| 價格 < BB_Lower | 跌入統計學 2.5% 極端超跌區 |
| RSI < 30 | 市場極度恐慌，散戶不計代價拋售 |

兩條件同時成立時，為「短線黃金冰點」，預期強勢反彈，觸發買進（`signal = 1`）。

### 賣出條件

| 條件 | 說明 |
|------|------|
| 價格 > BB_Upper | 衝破統計學 2.5% 極端高價區 |
| RSI > 70 | 市場極度亢奮貪婪，獲利了結壓力沉重 |

兩條件同時成立時，為「瘋狂頂點」，果斷停利賣出（`signal = -1`）。

### 適合股票

ETF（0050、006208、00878、0056）、大型金融股 — 具有箱型震盪與均值回歸特性的標的。

### 注意事項

- **高檔鈍化風險**：遇到強勢主力股時（如 2024-2026 年的 AI 概念股），股價會連續多天「黏在布林上軌外面」，RSI 天天卡在 85~90 之間（高檔鈍化）。策略會在剛起漲的前幾天就判定過熱而賣出，完美錯過後面的主升段。
- **抄底接刀風險**：權值崩盤時，股價會「沿著下軌一路向下摩擦」，RSI 連續幾週卡在低位。如果太早進去抄底，往往會「接到正在掉落的飛刀」。
- ✅ **已實裝優化**：大盤年線過濾（`MARKET_TREND_FILTER=true`，指數 < MA200 時跳過買進）— 實作於 `live_trader_multi.py:297` 與 `core/market_filter.py`
- ✅ **已實裝優化**：金字塔分批加碼（`PYRAMID_ENABLED=true`，200→400→600 股三梯次）— 實作於 `live_trader_multi.py:234-273`

### 參考原始碼

```python
class BollingerReverseStrategy(BackTest):
    def __init__(self, window=20, std_dev=2.0, rsi_period=5, rsi_low=30, rsi_high=70):
        self.window = window
        self.std_dev = std_dev
        self.rsi_period = rsi_period
        self.rsi_low = rsi_low
        self.rsi_high = rsi_high

    def calculate_rsi(self, prices):
        delta = prices.diff()
        gain = (delta.where(delta > 0, 0)).rolling(window=self.rsi_period).mean()
        loss = (-delta.where(delta < 0, 0)).rolling(window=self.rsi_period).mean()
        rs = gain / loss
        return 100 - (100 / (1 + rs))

    def trade(self, stock_price):
        if len(stock_price) < self.window:
            return 0
        close = stock_price['close']
        middle = close.rolling(self.window).mean().iloc[-1]
        std = close.rolling(self.window).std().iloc[-1]
        upper = middle + self.std_dev * std
        lower = middle - self.std_dev * std
        current_price = close.iloc[-1]
        rsi_series = self.calculate_rsi(close)
        rsi = rsi_series.iloc[-1] if not rsi_series.empty else 50
        if current_price < lower and rsi < self.rsi_low:
            return 1
        elif current_price > upper and rsi > self.rsi_high:
            return -1
        else:
            return 0
```

## 2. VWAP 偏離度反轉（VWAP）

**一句話**：股價偏離法人平均成本（VWAP）超過 1.5 倍標準差，且極短線情緒極端時，反向操作。

📺 **動畫演示**：[點擊觀看](https://dinchentech.github.io/tw-autotrader/vwap-animation.html) — 以走勢動畫解說買賣點邏輯

### 策略原理

VWAP（成交量加權平均價）代表市場所有資金真正的平均買進成本。散戶無法大幅影響 VWAP，只有大盤法人進出才會讓 VWAP 移動。當股價遠離 VWAP 超過一定倍數的標準差時，代表價格嚴重偏離法人的合理成本區，預期均值回歸。

數學定義：

| 元件 | 公式 | 意義 |
|------|------|------|
| VWAP | Σ(close × volume) / Σ(volume) | 市場真實平均成本 |
| Deviation | close − VWAP | 價格偏離法人的程度 |
| Std | STD(Deviation, 20) | 偏離度的波動標準差 |
| sigma_mult | 1.5（可調參數） | 極端門檻倍數 |
| RSI(5) | 100 − 100 / (1 + RS) | 極短線情緒指標 |

**AI 白話解說**：VWAP 不是單純把價格相加除以天數，而是把「每天的成交量」當作權重計算出的平均價（Σ(價格 × 成交量) / Σ 成交量）。它代表市場上所有資金真正的平均買進成本。一般散戶用幾張股票無法影響 VWAP，只有大盤主力、法人瘋狂倒貨或吃單時，VWAP 才會大幅移動。偏離度計算目前的股價距離主力的平均成本（VWAP）有多遠；`sigma_mult = 1.5` 設定 1.5 倍的標準差，當股價距離 VWAP 超過此門檻時，代表股價已經嚴重偏離了法人的合理成本區。程式刻意使用 `period=5` 的 RSI（一般常用 14），對短線恐慌或貪婪的情緒捕捉得非常極端且敏銳。

**買進邏輯**：股價不僅跌破了散戶的心理防線，甚至跌到了遠低於大盤法人平均成本（VWAP）的非理性低價，同時極短線恐慌到了極點。此時法人的護盤資金或左側交易主力隨時會進場護盤或抄底，讓股價彈回 VWAP（均值回歸）。

**賣出邏輯**：股價被市場熱情推到遠高於法人的平均成本區，短線過熱（RSI > 70），此時高檔獲利了結的賣壓（尤其是法人的調節賣單）會極其沉重，程式果斷在主力倒貨前先賣出（或進行融券放空）。

### 買進條件

| 條件 | 說明 |
|------|------|
| 價格 < VWAP − sigma_mult × Std | 股價遠低於法人平均成本（非理性低價） |
| RSI < 30 | 極短線恐慌到極點 |

預期法人護盤或左側交易資金進場，股價彈回 VWAP（`signal = 1`）。

### 賣出條件

| 條件 | 說明 |
|------|------|
| 價格 > VWAP + sigma_mult × Std | 股價遠高於法人平均成本（短線過熱） |
| RSI > 70 | 極短線貪婪到極點 |

預期高檔獲利了結賣壓湧現，在主力倒貨前賣出（`signal = -1`）。

### 適合股票

穩定金融股（富邦金、國泰金、兆豐金）— 與法人成本黏著度高，偏離後回歸機率大。

### 注意事項

- **✅ 回測已修正（2026-06-10）**：VWAP 改為滾動 20 期成交量加權計算（`Σ(close×volume)/Σ(volume)`），取代原本的 `close.mean()`。2881（富邦金）回測從 0 交易修正為 7 買 7 賣 +11.9%，2886（兆豐金）仍為 0 交易（20 期內偏離未達 1.5σ）。**實盤對接即時 VWAP 後，2886 也會正常觸發。**
- **實盤威力**：對接券商 API 取得即時 VWAP 後，對權值股盤中急殺反彈點的捕捉準確度極高。實盤中券商會提供當天即時的內建 VWAP，用來抓「台積電、聯發科」等權值股在盤中的開高走低低點、或是急殺後的反彈點，準確度非常驚人。
- 偏向逆勢策略，不適合強勢單邊行情。

### 參考原始碼

```python
def vwap_deviation_strategy(df: pd.DataFrame, sigma_mult=1.5, rsi_period=5, rsi_low=30, rsi_high=70) -> pd.DataFrame:
    df = df.copy()
    # VWAP = Σ(close × volume) / Σ(volume)，滾動 20 期
    df['VWAP'] = (df['close'] * df['volume']).rolling(window=20).sum() / df['volume'].rolling(window=20).sum()
    df['RSI'] = calculate_rsi(df['close'], rsi_period)
    df['Deviation'] = df['close'] - df['VWAP']
    df['Std'] = df['Deviation'].rolling(window=20, min_periods=1).std().fillna(0)
    df['signal'] = 0
    long_condition = (df['close'] < df['VWAP'] - sigma_mult * df['Std']) & (df['RSI'] < rsi_low)
    short_condition = (df['close'] > df['VWAP'] + sigma_mult * df['Std']) & (df['RSI'] > rsi_high)
    df.loc[long_condition, 'signal'] = 1
    df.loc[short_condition, 'signal'] = -1
    return df
```

## 3. 均線交叉（MA Cross）

**一句話**：短期平均成本突破中期平均成本時順勢買進；跌破時順勢賣出。

📺 **動畫演示**：[點擊觀看](https://dinchentech.github.io/tw-autotrader/ma_cross-animation.html) — 以走勢動畫解說買賣點邏輯

### 策略原理

利用兩條不同週期的移動平均線的相對位置變化，捕捉波段趨勢。快線（MA 9）對價格反應靈敏，慢線（MA 21）代表中期方向。當快線由下往上突破慢線（黃金交叉），為波段漲勢確立；反之則為多頭結束（死亡交叉）。

使用 `.shift(1)` 比對昨日與今日的相對位置，精確判定交叉瞬間。

數學定義：

| 元件 | 公式 | 意義 |
|------|------|------|
| MA_Fast | SMA(close, 9) | 短期平均成本 |
| MA_Slow | SMA(close, 21) | 中期平均成本（一個月） |
| Shift(1) | 取昨日資料 | 判斷交叉與否的基準 |

**AI 白話解說**：均線交叉策略是金融量化交易中最經典的「黃金交叉與死亡交叉」策略。快線（MA 9）因為天數短，對股價最近的漲跌反應非常靈敏，會緊貼著股價走；慢線（MA 21）代表過去約一個月的市場平均成交價格，移動較為緩慢平滑。程式碼中的 `.shift(1)` 代表「昨天的資料」，要判斷兩條線有沒有交叉，必須拿今天的相對位置和昨天的相對位置做對比。

**黃金交叉買進邏輯**：原本在下方落後的短期成本（快線），突然強力衝破了中期的平均成本（慢線）。這代表短線上的買盤力道極強，正在扭轉原本中期的趨勢。這通常是主力資金開始連續進場、拉抬股價的訊號，策略在此處判定「波段漲勢確立」，順勢買進。

**死亡交叉賣出邏輯**：短期的平均成本（快線）跌破了中期成本（慢線），代表短線追價力道枯竭，市場開始集體拋售，導致股價跌破一個月來的平均支撐。策略判定多頭行情結束，波段即將轉弱或轉空，果斷賣出套現。

### 買進條件（黃金交叉）

| 條件 | 說明 |
|------|------|
| 今日 MA_Fast > MA_Slow | 短期成本已超越中期成本 |
| 昨日 MA_Fast ≤ MA_Slow | 昨日尚未交叉 |

代表短線買盤力道極強，正在扭轉中期趨勢，順勢買進（`signal = 1`）。

### 賣出條件（死亡交叉）

| 條件 | 說明 |
|------|------|
| 今日 MA_Fast < MA_Slow | 短期成本已跌破中期成本 |
| 昨日 MA_Fast ≥ MA_Slow | 昨日尚未交叉 |

代表短線追價力道枯竭，多頭行情結束，果斷賣出（`signal = -1`）。

### 適合股票

半導體權值股（台積電 2330、聯發科 2454）— 龍頭股一旦發動大波段會走得很遠，均線交叉能完整參與漲幅。

### 注意事項

- **盤整期致命缺點**：股價橫向震盪時，快慢線頻繁糾纏交叉，產生連續假訊號（被「雙巴」），手續費與滑價會迅速侵蝕本金。當股票進入沒量、橫盤整理的箱型區間時，程式會陷入「今天金叉買進，明天馬上死叉停損；後天又金叉買回」的無限循環。
- **ATR 波動度過濾**（已內建）：策略會計算 ATR（平均真實波動幅度），當 ATR / 股價 < 0.5% 時判定為盤整期，跳過該次交叉訊號，避免被雙巴。可透過 `.env` 調整門檻：
  ```env
  MA_CROSS_ATR_THRESHOLD=0.005   # 門檻值（0.5%），調高 = 更嚴格的波動度要求
  ```
- **與布林策略的對比**：布林通道是「反轉抄底策略」，而均線交叉則是「順勢突破策略」。優點是如果台股再次發動像台積電或 AI 股那種一路上漲 3、4 個月的大行情，這個策略會在剛起漲時就金叉買進，並一路抱到波段頂點，直到趨勢反轉才死叉賣出，能賺到最肥美的一段利潤。
- 運算極輕量，單台低規主機即可同時跑多檔股票。

### 參考原始碼

```python
class MACrossStrategy(BackTest):
    def __init__(self, fast_period=9, slow_period=21, atr_period=14, atr_threshold=0.005):
        self.fast_period = fast_period
        self.slow_period = slow_period
        self.atr_period = atr_period
        self.atr_threshold = atr_threshold

    def trade(self, stock_price):
        if len(stock_price) < self.slow_period:
            return 0

        ma_fast = stock_price['close'].rolling(self.fast_period).mean()
        ma_slow = stock_price['close'].rolling(self.slow_period).mean()

        # 計算 ATR（平均真實波動幅度）— 用來過濾盤整期假訊號
        high = stock_price['high']
        low = stock_price['low']
        close = stock_price['close']

        prev_close = close.shift(1)
        tr = pd.concat([
            high - low,
            (high - prev_close).abs(),
            (low - prev_close).abs()
        ], axis=1).max(axis=1)
        atr = tr.rolling(self.atr_period).mean()

        # ATR 波動度過濾：ATR / 股價 < 門檻值 => 視為盤整，跳過訊號
        current_atr_ratio = atr.iloc[-1] / close.iloc[-1]
        is_volatile_enough = current_atr_ratio >= self.atr_threshold

        if ma_fast.iloc[-2] <= ma_slow.iloc[-2] and ma_fast.iloc[-1] > ma_slow.iloc[-1]:
            if is_volatile_enough:
                return 1
            else:
                return 0  # 波動不足，跳過

        elif ma_fast.iloc[-2] >= ma_slow.iloc[-2] and ma_fast.iloc[-1] < ma_slow.iloc[-1]:
            if is_volatile_enough:
                return -1
            else:
                return 0  # 波動不足，跳過

        else:
            return 0
```

## 4. 唐奇安通道突破（Breakout）

**一句話**：股價突破近一個月最高價時猜測主力進場，順勢買進；跌破近一個月最低價時猜測主力撤退，順勢賣出。

📺 **動畫演示**：[點擊觀看](https://dinchentech.github.io/tw-autotrader/breakout-animation.html) — 以走勢動畫解說買賣點邏輯

### 策略原理

唐奇安通道（Donchian Channel）以過去 N 日的最高價與最低價作為上下軌，勾勒出箱型整理區間。散戶的資金分散且無共識，只有大盤法人有能力將股價推破長期的天花板（N 日最高價），因此價格突破通道上軌即視為主力發動訊號。

使用前一日的通道值（`.shift(1)` 或 `iloc[-2]`），避免使用今日包含突破瞬間的資料造成前視偏差。

數學定義：

| 元件 | 公式 | 意義 |
|------|------|------|
| Donchian_High | MAX(high, N) | N 日（預設 20）市場最高價 |
| Donchian_Low | MIN(low, N) | N 日（預設 20）市場最低價 |

**AI 白話解說**：唐奇安通道以過去 20 天（約一個月）的最高價與最低價作為上下軌，勾勒出一個「箱型整理區間」，代表這檔股票近一個月來的市場天花板與地板。ATR（平均真實波動幅度，Average True Range）計算的是股票每天真正的震盪幅度（包含當天最高最低價差，以及跳空開高開低的缺口）。當 ATR 很小時，代表市場死寂、沒人在玩、波動極低（也就是「波動度擠壓」）；當 ATR 變大時，代表行情開始劇烈震盪，大資金進場在火拼了。

**買進邏輯**：股價強勢衝破了過去一個月來的最高點。這不是散戶買得起來的，代表有大盤、主力、法人正在大舉進場拉抬。同時要求市場整體的震盪幅度必須大於股價的 1%，確保「這場突破是帶著真槍實彈（高波動）的大行情」，而不是沒人要玩的冷門股因為一兩張買單意外創高。策略判定多頭波段正式發動，順勢買進上車。

**賣出邏輯**：股價跌破了過去一個月來的最底層支撐，且市場波動度不小。這代表主力資金已經撤退、棄守，甚至大盤法人正在集體倒貨。策略判定趨勢完全轉空，果斷賣出停損或停利。

**ATR 波動度過濾**：加入 ATR（Average True Range，平均真實波動幅度）計算，當 ATR/股價 低於門檻值時視為盤整，跳過買賣訊號，避免假突破。可透過 `.env` 設定 `BREAKOUT_ATR_PERIOD`（ATR 計算週期）和 `BREAKOUT_ATR_THRESHOLD`（波動門檻值，2025-06 後預設 0.02，此前為 0.01）。

### 買進條件

| 條件 | 說明 |
|------|------|
| 今日收盤價 > 昨日 Donchian_High | 股價強勢突破近一個月最高點 |

代表有大資金正在大舉進場拉抬，觸發買進順勢上車（`signal = 1`）。

### 賣出條件

| 條件 | 說明 |
|------|------|
| 今日收盤價 < 昨日 Donchian_Low | 股價跌破近一個月最低支撐 |

代表主力資金已撤退或棄守，果斷賣出停損或停利（`signal = -1`）。

### 適合股票

高價千金股與 AI 供應鏈（廣達 2382、緯穎 6669、世芯-KY 3661）— 這類飆股「不噴則已，一噴驚人」，適合順勢突破追進。

### 注意事項

- **假突破風險**：傳統突破策略最怕「今天突破、明天跌回」的假突破。已內建 ATR 波動度過濾（ATR/股價 ≥ 2% 才允許交易，2025-06 後預設值），可透過 `.env` 調整門檻：
  ```env
  BREAKOUT_ATR_THRESHOLD=0.02   # 門檻值（2%），調高 = 更嚴格的波動度要求
  ```
- 適合搭配移動停損（Trailing Stop）保護獲利，避免被主力洗盤甩轎。
- 盤整期完全無訊號，不會被雙巴，但也可能錯過緩漲型股票的初升段。

### 參考原始碼

```python
class BreakoutStrategy(BackTest):
    def __init__(self, lookback=20, atr_period=14, atr_threshold=0.01):
        self.lookback = lookback
        self.atr_period = atr_period
        self.atr_threshold = atr_threshold

    def trade(self, stock_price):
        if len(stock_price) < self.lookback + 1:
            return 0

        high = stock_price['high']
        low = stock_price['low']
        close = stock_price['close']
        current_price = close.iloc[-1]

        prev_close = close.shift(1)
        tr = pd.concat([
            high - low,
            (high - prev_close).abs(),
            (low - prev_close).abs()
        ], axis=1).max(axis=1)
        atr = tr.rolling(self.atr_period).mean()
        current_atr_ratio = atr.iloc[-1] / current_price
        is_volatile_enough = current_atr_ratio >= self.atr_threshold

        # 使用前一日的 Donchian 通道
        donchian_high = high.rolling(self.lookback).max().iloc[-2]
        donchian_low = low.rolling(self.lookback).min().iloc[-2]

        if current_price > donchian_high and is_volatile_enough:
            return 1
        elif current_price < donchian_low and is_volatile_enough:
            return -1
        else:
            return 0
```

## 5. Keep & Wait（DCA 低接策略）

### 核心邏輯

**不分析技術指標**，完全依價格乖離與紀律執行低接。適合波動大的高價股或 ETF 作為衛星持股：

📺 **動畫演示**：[點擊觀看](https://dinchentech.github.io/tw-autotrader/keep_wait策略說明.html) — 以走勢動畫解說買賣點邏輯

> 初始進場 → 跌 X% 再加碼 → 漲 Y% 停利賣出 → 冷卻後重來

### 買賣條件

| 動作 | 條件 | 說明 |
|------|------|------|
| **初始買進** | 啟動時立即買入 KW_INITIAL_SHARES 股 | 建立基本倉位 |
| **加碼買進** | 從均價跌 KW_ADD_DROP_PCT%（如 5%） | 最多 KW_MAX_ADDITIONS 次 |
| **停利賣出** | 從均價漲 KW_TP_TRIGGER_PCT%（如 15%） | 賣出 KW_TP_SELL_RATIO%（如 50%） |
| **冷卻期** | 賣出後 KW_COOLDOWN_DAYS 天不動作 | 避免短線反覆交易 |

### 參數說明

| 參數 | 預設值 | 單位 | 說明 |
|------|--------|------|------|
| `KW_INITIAL_SHARES` | 45 | 股 | 初始進場股數（依預算調整） |

| `KW_ADD_SHARES` | 45 | 股 | 每次加碼買入股數 |
| `KW_MAX_ADDITIONS` | 2 | 次 | 最多加碼次數（避免無限攤平） |
| `KW_TP_TRIGGER_PCT` | 15 | % | 從均價漲多少%停利 |
| `KW_TP_SELL_RATIO` | 50 | % | 停利時賣出比例（0=只通知不下單） |

## 6. 法人抬轎動能（Institutional Momentum）

> **一句話**：先找法人低檔吃貨的股票放觀察池，等它們出現動能再進場。

📺 **動畫演示**：[點擊觀看](https://dinchentech.github.io/tw-autotrader/法人動能策略說明.html) — 以走勢動畫解說篩選與買賣點邏輯

### 策略定位

不同於前面五種固定標的策略，法人抬轎動能是 **動態選股策略**：

- 不綁定特定股票，從全市場動態篩選
- 屬於 **Group 2（獨立資金池）**，不佔用 Group 1 的資金配置
- 預設關閉（`INST_MOM_CAPITAL=0`），需在 `.env` 設定本金才會啟用

### 策略原理

台股市場中，**投信（本土基金）** 與 **外資** 的行為可分成兩個階段：

1. **低檔吃貨**：法人在股價低檔默默累積籌碼（低量不跌、有量無價、下影線、法人買超轉正）
2. **拉抬出貨**：籌碼夠了開始拉抬，股價創新高、法人單日大買超

策略 6 的核心升級是 **兩階段篩選** — 先找有低吃紀錄的股票，再等它們出現動能才進場。這解決了原本動能策略最大的痛點：追高買到「假突破、真出貨」的股票。

```
每週：掃描法人低吃訊號 → 建立觀察池
每日：檢查觀察池內股票是否出現動能 → 有訊號隔日進場
```

## 附錄 A：獲利滾入本金（Profit Rolling）

系統支援**定期將已實現獲利自動滾入本金**，讓資金複利成長，而非閒置在帳戶中。

### 參數設定

| 參數 | 預設值 | 說明 |
|------|--------|------|
| `PROFIT_ROLL_MONTHS` | `5` | 每 N 個月滾入一次（0 = 每次賣出立即滾入） |
| `PROFIT_ROLL_PERCENTAGE` | `100` | 滾入比例（%），100 = 全部獲利滾入 |

```env
PROFIT_ROLL_MONTHS=5
PROFIT_ROLL_PERCENTAGE=100
```

> ⚠️ **重要說明**：
> - `PROFIT_ROLL_MONTHS` 是**時間間隔**（每幾個月滾入一次），**不是倍數**（不是「獲利乘以 N 倍」）。
> - `M=0` 表示**每次賣出立即滾入**已實現獲利，無等待期。
> - `M=5` 表示**每 5 個月滾入一次**（預設值），兼顧複利效果與風險控制。
> - 使用方式：`python simulate_portfolio.py --profit-roll-months 5 --profit-roll-percentage 1.0`

### 運作機制

1. **觸發條件**：`M=0` 時每次賣出立即滾入；`M>0` 時每 `M` 個月檢查一次（從初始日期起算）
2. **滾入金額**：`已實現損益 × P%`（已賣出股票的淨利潤）
3. **資金來源**：僅使用帳戶內資金，不追加外部資金
4. **資金分配**：滾入的資金加入 `general_cash`，可投資所有策略標的

### 範例說明

假設：`INITIAL_CAPITAL=500,000`、`PROFIT_ROLL_MONTHS=5`、`PROFIT_ROLL_PERCENTAGE=100`

| 時間 | 已實現獲利 | 滾入金額 | 新本金 |
|------|-----------|---------|--------|
| 第 5 個月 | +NT$50,000 | NT$50,000 | NT$550,000 |
| 第 10 個月 | +NT$80,000 | NT$80,000 | NT$630,000 |
| 第 15 個月 | +NT$120,000 | NT$120,000 | NT$750,000 |

> 💡 **複利效應**：每 5 個月滾入一次，資金穩定複利成長。長期下來報酬率差異可觀。
>
> ⚠️ **注意**：僅限已賣出（已實現）的獲利才會滾入。未賣出的浮動盈虧不計入。

## 7. 風險管控系統（RiskManager）

所有 Group 1 策略共用同一套風險控管，定義在 `core/risk_manager.py`，在 `live_trader_multi.py` 主循環中每筆交易前檢查。

### 控管項目

| 機制 | 參數（`.env`） | 預設值 | 說明 |
|------|---------------|--------|------|
| **每日交易次數上限** | `MAX_DAILY_TRADES` | 3 | 同一天同一檔股票最多交易 N 次（含買+賣）。資金充裕時（剩餘資金 > CAPITAL_CONTROL_LINE%）不限制。 |
| **每日虧損上限** | `MAX_DAILY_LOSS` | 0.05（5%） | 當日累計虧損超過此比例時，當日停止所有買進。 |
| **漲跌停過濾** | 內建 | 10% | 股價觸及漲停或跌停時跳過該筆交易，避免買在最高點或賣在最低點。 |
| **資金控制線** | `CAPITAL_CONTROL_LINE` | 30（%） | 剩餘資金低於此比例時才開始限制每日交易次數。資金充裕時不卡交易。 |
| **個股配置上限** | `PC_xxx` 的 `alloc` | 各股自訂 | 每檔股票累計買入金額不得超過 `TOTAL_CAPITAL × alloc%`。超過 70% 時 Telegram 提醒。 |
| **每月預算上限** | `PC_xxx` 的 `monthly_budget` | 0（不限） | 每檔股票每月最多買入金額。超過後當月不再買進。 |
| **大盤年線過濾** | `MARKET_TREND_FILTER` | true | 大盤指數跌破 MA200 時，跳過所有策略的買進訊號。資料源：TWSE 公開 API。 |

### 運作流程

```
收到買進訊號
  ↓
✓ 風險管理器檢查（每日次數 / 虧損上限 / 漲跌停）
  ↓
✓ 每月預算檢查（monthly_budget）
  ↓
✓ 個股資金上限檢查（alloc%）
  ↓  (超過 70% 發 Telegram 警告)
✓ 大盤年線過濾（MARKET_TREND_FILTER）
  ↓
✓ 庫存確認（賣出時檢查是否持有足夠股數）
  ↓
執行下單
```

任一站未通過則跳過該筆交易，不影響其他策略運作。

### 設定範例

```env
MAX_DAILY_TRADES=3          # 每日最多交易次數
MAX_DAILY_LOSS=0.05         # 每日最大虧損 5%
CAPITAL_CONTROL_LINE=30     # 資金低於 30% 才開始限制交易次數
MARKET_TREND_FILTER=true    # 啟用大盤年線過濾

# 個股資金上限 = TOTAL_CAPITAL × alloc%
TOTAL_CAPITAL=500000
PC_2330={"strategy":"ma_cross","alloc":15,...}  # 上限 = 500k × 15% = NT$75,000
```

## 8. 重要觀念：策略交易 vs 買入持有

> 很多投資人看到回溯報告的比較表後會問：「策略績效還輸給買入持有，那直接用買入持有不就好了？」

這個問題非常關鍵，以下用實際數據說明。

### 2024-2025 策略 vs 買入持有（50萬一筆資金）

| 比較項目 | 策略交易 | 買入持有 | 差異 |
|----------|---------|---------|------|
| 終值 | NT$681,791 | NT$866,867 | NT$-185,076 |
| 報酬率 | +36.4% | +73.4% | -37.0% |

> 📊 本次更新（2026-06-10）：VWAP 修正為滾動 20 期成交量加權計算後，2881（富邦金）從 0 交易改為 7 買 7 賣 +11.9%，整體組合報酬率由 +32.9% 提升至 +36.4%。

**策略落後買入持有的三個主要原因：**

1. **VWAP 2886（兆豐金）20 期內仍無訊號**：2881 修正後已有 7 買 7 賣 +11.9%，但 2886 在 20 期滾動視窗內偏離度未達 1.5σ 門檻，回測仍為 0 交易。**實盤**使用即時 VWAP 資料後預期會正常觸發。

2. **2454（聯發科）被均線假訊號雙巴**：2454 兩年漲了 +59%，但策略賠了 -6% — 14 次買進 14 次賣出、進進出出反被洗掉獲利。這是均線交叉在盤整期的天敵。

3. **策略本質在大多頭必然落後**：策略會停利出場 → 現金閒置 → 等待下次訊號，必然錯過一部分漲幅。買入持有從不賣出，在大多頭當然全數吃下。

### 那什麼時候策略會贏？

請看以下假設情境：

| 市場環境 | 買入持有 | 策略交易 | 誰勝出？ |
|---------|---------|---------|---------|
| 🟢 **大多頭**（如 2024-2025） | +73.4% | +36.4% | ❌ 策略落後 |
| 🟡 **高檔震盪**（如 2024-2025 中的聯發科） | +59% | -6%（被雙巴） | 策略不一定贏 |
| 🔴 **空頭/崩盤**（如 2022, 2008） | -20%~-50% | -3%~-10%（空手避開） | ✅ 策略大勝 |
| 🔵 **區間盤整** | 0%~+5% | +5%~+15%（低買高賣） | ✅ 策略大勝 |

散戶最大問題不是在多頭賺不夠多，而是：
- 多頭末段忍不住追高（買在最高點）
- 空頭來臨時死抱不賣（虧 30-50% 才停損）
- 沒有紀律的進出規則（漲了怕、跌了更怕）

### 策略的真正價值

| 面向 | 買入持有 | 策略交易 |
|------|---------|---------|
| 🎯 **多頭年** | 報酬最高 | 報酬次之（但已獲利了結） |
| 🛡️ **空頭年** | 跟著大跌 | 有紀律停損/空手，守住本金 |
| 😱 **人性考驗** | 靠信仰抱股（多數人做不到） | 程式自動執行，克服貪婪恐懼 |
| 📊 **報酬穩定度** | 波動劇烈（標準差大） | 波動較小（夏普值較高） |
| 💡 **勝率** | 只要選對股，長期必勝 | 依照策略特性，約 30-80% |

**一句話總結**：
> 買入持有的前提是「選對股票且抱得住」。策略交易則是「無論選對或選錯，都有紀律地進出」。

如果你有把握：
- 選到 2330、0050 這種長期向上的標的
- 在大跌 30% 時依然抱得住、甚至加碼

那買入持有確實是最佳策略。

但如果你：
- 不確定何時該停利
- 大跌時會恐慌賣在最低點
- 想讓程式自動管理、不用每天盯盤

那策略交易更有保障。

### 最簡單的做法：100% 交給策略

不需要煩惱「該買哪檔股票」、「該不該停利」、「現在是不是高點」。**全部資金按配置比例投入，讓四個策略自動管理**：

```
┌─────────────────────────────────────┐
│  總資金 NT$500,000                    │
│                                       │
│  100% 由 live_trader_multi.py 管理     │
│  ├─ Bollinger 40% → ETF 逆勢抄底       │
│  ├─ MA Cross   25% → 權值股順勢波段    │
│  ├─ VWAP       20% → 金融股逆勢存股    │
│  └─ Breakout   15% → 飆股突破追進      │
│                                       │
│  不用選股、不用盯盤、不用判斷多空       │
│  資金到了每月預算上限自動停            │
│  大盤跌破年線全部買進暫停              │
└─────────────────────────────────────┘
```

選股已經內建在 `PORTFOLIO` 設定中（0050、2330、2382、2881 等 8-12 檔），你只需要決定總資金和每月預算，剩下全部交給程式。選股時可以多問幾家 AI 的建議（如 ChatGPT、Claude、Gemini 等），交叉比對後再決定最終的 PORTFOLIO 組合。

以上Group 1四個策略正好互補，涵蓋了不同的交易情境與市場狀態：

| 策略 | 類型 | 核心邏輯 |
|------|------|---------|

| VWAP Deviation（VWAP 偏離） | 逆勢 | 抓法人成本極端偏離 |
| Breakout（唐奇安突破） | 順勢 | 主力追價突破 |
| Bollinger Reverse（布林反轉） | 逆勢 | 恐慌抄底、貪婪逃頂 |

### 自動化交易的核心價值

> 自動化交易的目標不是年年打敗大盤，而是在各種市場環境下都能穩定發揮：

- **📉 空頭時少賠**：2022 年那種 -20%~-50% 的崩盤，策略空手避開，定期定額照樣跟著跌
- **📊 盤整時賺價差**： yearly +5%~+15% 低買高賣，抱著不動是 0%
- **😴 不用盯盤**：程式自動跑，克服人性追高殺低
- **📈 多頭也跟得上**：2024-2025 兩年 +36.4%（50 萬一筆資金），雖然落後大盤 +73.4% 但不缺席

**一句白話**：行情好的時候你沒漏掉太多，行情差的時候你睡得著覺。

## 附錄：盤的種類與策略對應

不同盤勢有各自適合與不適合的策略，理解盤勢類型才能妥善配置：

| 盤勢 | 特徵 | 適合策略 | 不適合 |
|:---|:---|:---|:---|
| **趨勢盤** | 一直漲或一直跌，不回檔 | MA Cross（追趨勢）、Breakout（突破）、法人動能 | Bollinger（抄底會被軋） |
| **盤整盤** | 上下區間來回，沒方向 | Bollinger（低買高賣）、VWAP（偏離反轉） | 法人動能（掃不到創新高標的） |
| **震盪盤** | 急漲急跌但很快拉回 | VWAP（乖離過大反轉）、Bollinger | Breakout（假突破陷阱多） |
| **崩跌盤** | 連續大跌，躲都來不及 | keep_wait（慢慢低接，等復甦） | Breakout、法人動能（一買就停損） |
| **噴出盤** | 短線急拉，跳空不補 | Breakout（追強勢股最賺）、法人動能（吃外資買超最甜） | Bollinger（跌不下來，根本沒買點） |

**行話小整理**：

| 說法 | 意思 |
|:---|:---|
| 多頭/牛市 | 長期看漲 |
| 空頭/熊市 | 長期看跌 |
| 驚驚漲 | 天天懷疑但天天漲（最磨人） |
| 死魚盤 | 整天幾乎沒波動 |
| 多空雙巴 | 漲上去追就跌，跌下來空就漲 |

所以沒有「最強策略」，只有**選對盤的配置** — 這也是 `live_trader_multi.py` 同時掛 MA Cross + Bollinger + VWAP 三種策略的原因，不同盤至少有一兩個在賺。

### 附錄 : 技術分析的核心指標

| 指標 | 白話解讀 | 典型用法 |
|------|----------|----------|
| **K線圖** | 用紅綠蠟燭線表示股價漲跌 | 觀察多空力量對比 |
| **均線（MA）** | 最近N天平均價格線，即「平均成本線」 | 股價站上均線表示趨勢偏強 |
| **成交量** | 當天成交的總張數（1張=1,000股） | 放量上漲才是真漲 |
| **MACD** | 像「油門力道」——紅柱變長=加速上漲 | 黃金交叉視為買進訊號 |
| **KD** | 像「情緒溫度計」，看價格在近期區間的位置 | K值>80超買，<20超賣 |
| **RSI** | 統計漲跌力道，像「心率」 | 數值>70超買，<30超賣 |
| **布林通道** | 均線上下加標準差範圍，像「橡皮筋」 | 觸及上軌可能回調，觸及下軌可能反彈 |
| **乖離率（BIAS）** | 股價與移動平均線的偏離程度 | 正乖離過大→漲太多，負乖離過大→跌太深 |
| **OBV** | 上漲加成交量、下跌減成交量，像「資金腳步聲」 | 價格與OBV走勢背離可能預示變盤 |

**⚠️ 關鍵提醒**：技術指標不應單獨使用，需要多個指標相互印證。台股入門建議先掌握K線、均線與成交量。

### 附錄：標準差補充說明

標準差告訴你：股價在過去一段時間內，偏離平均價格的程度有多大。
以最近 5 天的收盤價為例：
收盤價：100、102、101、103、105
平均 = 102.2

標準差就是計算「每一天的價格與平均的差距」，然後取平均（經過平方開根號）。

### 附錄：RSI 指標補充說明

RSI（相對強弱指標，Relative Strength Index）是一種用來評估市場「買賣雙方力道強弱」的動量指標。它透過計算特定期間內股價漲跌幅的大小與速度，來判斷股價是否處於過熱（超買）或過冷（超賣）的狀態。

**核心數值與判讀邏輯**：RSI 數值介於 0 到 100 之間，最常用的計算天數為 14 天：
- **50（中軸）**：多空分水嶺，大於 50 代表買方較強，小於 50 代表賣方較強。
- **70 以上（超買區）**：代表市場過熱（漲多），追高風險較大，可能隨時會出現回檔下跌。
- **30 以下（超賣區）**：代表市場過冷（跌深），常被視為具備跌深反彈的機會，或者是空頭力道減弱的訊號。

### 附錄：ATR（Average True Range）指標補充說明

ATR（平均真實波動幅度，Average True Range）是由威爾斯·威爾德（J. Welles Wilder）在 1978 年提出的技術指標，用來衡量市場的**波動程度**，**不預測方向**。

#### 一句話理解
> ATR 告訴你「這檔股票最近一天平均會波動多少錢」，數字越大代表越劇烈、越有人在玩。

#### 計算方式

ATR 的基礎是「真實波動幅度」（True Range, TR），定義為以下三者的**最大值**：

```
TR = max( high − low , |high − prev_close| , |low − prev_close| )
```

| 組成 | 公式 | 意思 |
|------|------|------|
| ① 當日高低差 | high − low | 今天盤中震了多大 |
| ② 向上跳空 | |high − prev_close| | 今天最高比昨天收盤高多少 |
| ③ 向下殺低 | |low − prev_close| | 今天最低比昨天收盤低多少 |

> 最後：**ATR = TR 的 N 日移動平均**（本系統預設 N=14）

#### 在本系統中的用途

ATR 被用來過濾**盤整期的假訊號**。當 ATR / 股價 低於門檻值時，代表市場死氣沉沉，此時產生的買賣訊號可靠性低，策略會跳過不交易。

| 策略 | ATR 門檻 | 門檻值 | 用途 |
|------|---------|--------|------|
| **MA Cross**（均線交叉） | ATR / 股價 ≥ 0.5% | `MA_CROSS_ATR_THRESHOLD=0.005` | 跳過盤整期的黃金/死亡交叉假訊號 |
| **Breakout**（突破交易） | ATR / 股價 ≥ 2% | `BREAKOUT_ATR_THRESHOLD=0.02` | 過濾沒動能的冷門股假突破 |

#### 實戰意義

| ATR 狀態 | 市場含義 | 策略行為 |
|----------|---------|---------|
| **ATR 很小**（低於門檻） | 盤整、沒人玩、波動死亡 | ❌ 跳過訊號，避免被雙巴 |
| **ATR 正常或偏大**（高於門檻） | 有主力在操作、波動健康 | ✅ 正常執行買賣判斷 |

> **簡單記法**：ATR 就是股票的「震度計」——規模太小（低於門檻）的地震不值得跑，規模夠大再動作。

### 附錄: KD線 指標

股票的 KD 線（隨機指標）可以用圖示來畫出它的走勢與買賣訊號。KD 指標由兩條線組成，K 線反應比較快（快線），D 線反應比較慢（慢線）。這兩條線會在 0 到 100 之間上下波動，幫助我們看出市場是太熱還是太冷。

#### 🧮 詳細計算公式

KD 線的計算核心是 **RSV（未成熟隨機值）**。以市場最常用的 **9 天期 KD（9, 3, 3）** 為例，計算步驟如下：

#### 1️⃣ 步驟一：計算 RSV
RSV 用來衡量「今天的收盤價」在「過去 9 天內」屬於高點還是低點。
\[\text{RSV} = \frac{\text{今天的收盤價} - \text{過去 9 天內的最低價}}{\text{過去 9 天內的最高價} - \text{過去 9 天內的最低價}} \times 100\]

#### 2️⃣ 步驟二：計算 K 值（快線）
將舊的 K 值與今天的 RSV 進行平滑混合。
\[\text{今天的 K 值} = \left( \frac{2}{3} \times \text{昨天的 K 值} \right) + \left( \frac{1}{3} \times \text{今天的 RSV} \right)\]

#### 3️⃣ 步驟三：計算 D 值（慢線）
將舊的 D 值與今天的 K 值進行平滑混合。
\[\text{今天的 D 值} = \left( \frac{2}{3} \times \text{昨天的 D 值} \right) + \left( \frac{1}{3} \times \text{今天的 K 值} \right)\]

> 💡 **小提示**：如果股票剛上市、沒有「昨天的 K, D 值」，市場慣例會**直接以 50 帶入**作為初始值。

### 附錄: MACD 指標完整說明

MACD 全名為 **指數平滑異同移動平均線**（Moving Average Convergence Divergence），由 Gerald Appel 於 1970 年代提出，是技術分析中最常使用的**趨勢動能指標**。

---

#### 一、MACD 的三大組成

| 名稱 | 公式 | 白話解釋 |
|------|------|----------|
| **DIF（快線）** | EMA(12) – EMA(26) | 短天期均線 減 長天期均線。數值為正表示短期強於長期（多頭）；為負表示短期弱於長期（空頭） |
| **DEA(慢線)** | DIF 的 9 日 EMA（平滑） | DIF 的移動平均，用來確認 DIF 的趨勢，減少雜訊 |
| **MACD 柱狀體** | DIF – DEA | 兩線差值，用「柱狀」呈現。 >0 為紅柱（多頭動能），<0 為綠柱（空頭動能） |

> 📌 **預設參數**：(12, 26, 9)  
> - 12：短期 EMA 的週期  
> - 26：長期 EMA 的週期  
> - 9：DIF 的平滑週期（用來計算 DEA）

---

#### 二、白話計算流程（不必死記）

1. **計算兩條 EMA**  
   收盤價的 12 日 EMA 與 26 日 EMA。

2. **相減得到 DIF**  
   DIF = EMA(12) – EMA(26)。  
   - DIF > 0：短期平均在長期平均之上（多頭排列）。  
   - DIF < 0：短期平均在長期平均之下（空頭排列）。

3. **平滑 DIF 得到 DEA**  
   將 DIF 再取 9 日 EMA 得到 DEA。

4. **計算柱狀體**  
   柱狀值 = DIF – DEA。  
   - 正值（紅柱）：DIF 在 DEA 之上，動能向上。  
   - 負值（綠柱）：DIF 在 DEA 之下，動能向下。

---

#### 三、如何看圖（以台股常用軟體為例）

- **DIF**：一條較快速、波動較大的線（常設為藍色或白色）。  
- **DEA**：一條較平滑、跟隨在後的線（常設為黃色或紅色）。  
- **柱狀體**：以零軸為中心，向上為紅（或正數），向下為綠（或負數）。

---

#### 四、主要買賣訊號

##### 1. 黃金交叉（買進訊號）

> **DIF 由下往上穿越 DEA**

- 表示短期動能轉強，趨勢可能由空翻多或多頭加速。
- 建議搭配**零軸位置**判斷：
  - **零軸以上的黃金交叉** → 強勢多頭中的加碼點（可信度高）。
  - **零軸以下的黃金交叉** → 空頭反彈，可能是短多或反轉試探。

##### 2. 死亡交叉（賣出訊號）

> **DIF 由上往下跌破 DEA**

- 表示短期動能轉弱，趨勢可能由多翻空或空頭加速。
- 零軸以下的死亡交叉 → 空頭確立；零軸以上的死亡交叉 → 多頭回檔，可先減碼。

##### 3. 柱狀體變化（動能強弱）

- **紅柱變長**：DIF 與 DEA 張口擴大，動能增強，趨勢持續。
- **紅柱縮短**：DIF 上升速度減緩，動能衰減，可能準備轉折。
- **綠柱變長**：空頭動能增強。
- **綠柱縮短**：空頭動能衰減，可能準備反彈。

---

#### 五、背離（最重要的進階用法）

背離是指 **股價走勢** 與 **MACD 走勢** 不一致，經常預示趨勢反轉。

##### 頂背離（賣出訊號）
- **股價** 創新高（一波比一波高）。
- **MACD（通常是 DIF 或柱狀體高點）** 沒有創新高，甚至反向走低。
- 代表上漲動能跟不上股價，容易反轉下跌。

##### 底背離（買進訊號）
- **股價** 創新低（一波比一波低）。
- **MACD（DIF 或柱狀體低點）** 沒有創新低，反而抬高。
- 代表下跌動能減弱，可能反彈上漲。

### 附錄: MA 與 EMA

#### MA 與 EMA 差異完整說明

在技術分析中，移動平均線是最基礎且最重要的工具之一。最常見的兩種移動平均線為 **MA（簡單移動平均線）** 與 **EMA（指數移動平均線）**。雖然兩者都用來平滑股價數據、顯示趨勢方向，但它們的計算邏輯與特性有明顯差異。
---

#### 一、基本定義

| 名稱 | 全名 | 中文譯名 |
|------|------|----------|
| **MA** | Simple Moving Average | 簡單移動平均線 |
| **EMA** | Exponential Moving Average | 指數移動平均線 |

---

#### 二、計算方式差異

##### MA（簡單移動平均）

> 將過去 N 天的收盤價加總，除以 N，每一天的權重相同。

MA(N) = (P₁ + P₂ + ... + Pₙ) / N

**白話舉例**（N=5）：
最近 5 天收盤價：100、102、101、103、105  
MA(5) = (100+102+101+103+105) / 5 = **102.2**  
→ 每天影響力一樣。

---

##### EMA（指數移動平均）

> 給予近期價格較高的權重，越靠近今天的價格影響越大；歷史價格影響力呈指數衰減。

EMA(今日) = (P今日 × k) + [EMA(昨日) × (1 - k)]

**白話舉例**（N=5，k=2/6≈0.333）：
若昨日 EMA=102，今日價格=105，則  
EMA(今日) = 105×0.333 + 102×0.667 ≈ 103  
→ 今日價格權重約 33%，遠高於 MA 中的 20%。

---

#### 三、特性對比表

| 特性 | MA | EMA |
|------|-----|-----|
| **權重分配** | 所有天數權重相等 | 近期權重較高，遠期權重較低 |
| **反應速度** | 較慢（落後實際價格） | 較快（較貼近近期價格） |
| **平滑程度** | 較平滑（曲線弧度較大） | 較不平滑（曲線較貼近價格波動） |
| **計算複雜度** | 簡單（加總除 N） | 略複雜（需昨日 EMA 值） |
| **滯後性** | 明顯 | 較不明顯 |
| **跳空缺口影響** | 均勻反映 | 近期缺口影響較大 |

### Q5：投資組合（選股/比例）本身如何幫助避免策略擁擠？

**簡答：你的四套策略綁定四種完全不同類型的股票，它們的交易對手根本不重疊，這本身就是最有效的擁擠防護。**

#### 策略擁擠的本質

策略擁擠指的是「太多人用同一套邏輯在同一批股票上交易」，導致訊號邊際效益遞減：

- 布林抄底大家都在同一條下軌接 → 跌破軌道後彈不起來（沒人抬轎）
- 突破大家都在同一個價位買 → 跳空開高沒有回檔加碼機會

#### 你的組合為什麼不容易擁擠

你的五策略不僅邏輯不同，連**交易的股票都完全不同**：

| 策略 | 綁定股票類型 | 主要交易對手 | 市場參與者規模 |
|------|-------------|-------------|--------------|
| 布林通道反轉 | ETF（0050/006208/00878/0056） | 所有持有 ETF 的投資人、法人 | 🔴 最大（數千億級） |
| VWAP 偏離度反轉 | 金融股（2881/2882/2886） | 外資、投信、壽險大戶 | 🔴 很大（法人持倉極重） |
| 均線交叉 | 半導體權值股（2330） | 外資、自營商、大戶散戶 | 🟡 中等（流動性極深） |
| 唐奇安突破 | AI 供應鏈（2382/6669/3661） | 主力、法人大戶、當沖客 | 🟢 最小（特定資金鎖定） |
| **Keep & Wait** | **高價波動股（2454）** | **散戶中實戶** | ![🟢] 極小（特定標的低頻交易） |

**關鍵洞察**：一個在 0050 上用量化抄底的法人，跟一個在 3661 上追 AI 突破的散戶大戶，**根本不是同一批市場參與者**。即使兩者都用布林通道或用 VWAP，他們交易的股票不重疊，就不會互相擁擠。

#### 具體擁擠保護機制

1. **股票池隔離**：每套策略有專屬 2~3 檔股票，不會發生四套策略同時搶同一檔股票的情況
2. **資金配額隔離**：`ALLOC_*` 確保每套策略的預算分開，不會因為某一策略表現好就吃光全部資金（以 50 萬一筆資金為例）：
   ```env
   ALLOC_BOLLINGER=35     # ETF 35% → NT$175,000
   ALLOC_MA_CROSS=15      # 權值股 15% → NT$75,000
   ALLOC_BREAKOUT=15      # 飆股 15% → NT$75,000
   ALLOC_VWAP=20          # 金融股 20% → NT$100,000
   ALLOC_KEEP_WAIT=15     # DCA 低接 15% → NT$75,000
   ```
3. **策略類型對沖**：即使所有策略都同時碰到交易的瓶頸期，順勢（MA、Breakout）與逆勢（Bollinger、VWAP）兩種完全相反的交易邏輯也會互相抵銷擁擠風險 — 在大漲時逆勢策略不進場（或出場），在大跌時順勢策略不出手：
   ```
   市場狀態        順勢策略行為   逆勢策略行為   整體風險
   ───────────    ──────────    ──────────    ────
   🟢 大多頭       ✅ 頻繁買進    ❌ 空手或賣出   對沖
   🔴 大空頭       ❌ 停損空手    ✅ 開始進場抄底  對沖
   🟡 盤整         ❌ ATR 過濾    ❌ 偏離度不到   同時空手
   ```

#### 跟 Q1 的差別

- **Q1** 回答的是「有人故意針對我的策略陷害我嗎？」— 答案是零股太小，沒人會做這種虧本生意。
- **Q5** 回答的是「就算沒人針對我，策略自然老化（太多人用同一套）會不會失效？」— 答案是你的投資組合配置（不同類型股票綁定不同策略）讓四套策略的交易對手不重疊，自然老化速度比 all-in 單一策略慢得多。

#### 一句話總結

> 你的最大擁擠防火牆不是參數多厲害，而是**四套策略配上四類股票** — 它們的交易對手從散戶、ETF 投資人、外資到主力全不重疊，這比任何參數優化都更難被市場磨平。

## 9. `.env` 參數與資金池設定範例

本系統支援將資金劃分為兩個完全獨立的「資金池」，並可透過 `.env` 環境變數彈性配置不同的投資模式（定額存股 vs 單筆重壓）。

### 9.1 兩大資金池機制說明

| 資金池 | 負責程式 | 資金來源 | 配置方式 | 適合對象 |
|------|---------|---------|---------|---------|
| **Group 1<br>(固定標的)** | `live_trader_multi.py` | `TOTAL_CAPITAL`<br>（或每月 `DCA_AMOUNT`） | 透過 `PC_<代號>` 獨立分配每檔股票的資金比例 (`alloc`) 與策略參數。 | 長期核心資產、想穩健累積 ETF 與權值股的投資人。 |
| **Group 2<br>(法人動能)** | `live_trader_multi.py` 內建呼叫 | `INST_MOM_CAPITAL` | 獨立 50 萬以上的單獨資金，不吃 Group 1 的額度。全市場動態選股，最多持有 3 檔。 | 有大筆閒置資金、想承受較大波動以換取超額報酬（>30%）的投資人。 |

> 💡 **互不干擾**：Group 1 跟 Group 2 的資金與交易額度是**分開計算**的。Group 2 虧損不會影響 Group 1 的交易，反之亦然。

### 9.2 策略參數說明

在 `.env` 中，我們使用 V1.11 的 JSON 格式設定每檔股票。通用欄位如下：

- **`strategy`**：使用哪種策略（bollinger, vwap, ma_cross, breakout, keep_wait）
- **`alloc`**：資金佔比（%）。例如 `alloc:30` 代表分配 `TOTAL_CAPITAL` 的 30% 給這檔股票。
- **`monthly_budget`**：每月預算上限（金額）。通常在 DCA 模式下使用，Lumpsum 模式不需設定。

各策略的專屬參數（填入 JSON 內）：

| 策略 | 專屬參數 | 說明 |
|------|---------|------|
| **bollinger** | `window`, `std_dev`, `rsi_period` | 均線天數、標準差倍數（決定通道寬度）、RSI 計算天數 |
| **vwap** | `sigma_mult`, `rsi_period` | VWAP 偏離的標準差倍數、RSI 天數 |
| **ma_cross** | `fast_period`, `slow_period` | 快慢線均線天數（如 9, 21） |
| **breakout** | `lookback` | 突破回溯天數（如 20 代表突破近 20 日高點進場） |
| **keep_wait**| 無 | 使用全域的 `KW_ADD_DROP_PCT`, `KW_TP_TRIGGER_PCT` 等低接參數 |

### 9.3 設定範例 A：小資族每月 2 萬 DCA 定期定額

**情境**：每月固定拿 2 萬元出來投資，不玩法人動能，專注累積核心資產。
- 0050（布林反轉）分配 35%
- 00878（布林反轉）分配 20%
- 2881（VWAP 金融股）分配 20%
- 2330（均線交叉）分配 25%

**`.env` 寫法：**
```env
# --- 資金模式設定 ---
TOTAL_CAPITAL=0               # DCA 模式不看總資金
DCA_AMOUNT=20000              # 啟用每月定額模式
INST_MOM_CAPITAL=0            # 關閉 Group 2 法人動能

# --- Group 1 固定標的配置 (JSON 格式) ---
PC_0050={"strategy":"bollinger","alloc":35,"window":20,"std_dev":2.0,"rsi_period":5,"monthly_budget":7000}
PC_00878={"strategy":"bollinger","alloc":20,"window":20,"std_dev":2.0,"rsi_period":5,"monthly_budget":4000}
PC_2881={"strategy":"vwap","alloc":20,"sigma_mult":1.5,"rsi_period":5,"monthly_budget":4000}
PC_2330={"strategy":"ma_cross","alloc":25,"fast_period":9,"slow_period":21,"atr_threshold":0.005,"monthly_budget":5000}
```
*(註：`monthly_budget` 確保每檔股票不會把其他人的扣款預算吃光)*

### 9.4 設定範例 B：主力大戶 50 萬 Lumpsum + 50 萬法人動能

**情境**：手上有 100 萬資金，一半放長期核心持股（Group 1），一半拿來做高爆發的動態選股（Group 2）。
- Group 1（50 萬）：0050 (40%)、2330 (30%)、2454 低接 (30%)
- Group 2（50 萬）：交給法人抬轎動能全自動篩選

**`.env` 寫法：**
```env
# --- 資金模式設定 ---
TOTAL_CAPITAL=500000          # 啟用單筆資金模式，總額 50 萬
DCA_AMOUNT=0                  # 關閉 DCA
INST_MOM_CAPITAL=500000       # 啟用 Group 2 法人動能，獨立本金 50 萬

# --- Group 1 固定標的配置 (JSON 格式) ---
# 單筆資金模式由 alloc 控制上限，不用設 monthly_budget
PC_0050={"strategy":"bollinger","alloc":40,"window":20,"std_dev":2.0,"rsi_period":5}
PC_2330={"strategy":"ma_cross","alloc":30,"fast_period":9,"slow_period":21,"atr_threshold":0.005}
PC_2454={"strategy":"keep_wait","alloc":30}

# --- Group 2 法人動能參數 ---
# 這些是回測找出的最佳預設值，寫在 .env 讓主程式讀取
INST_FISH_SCORE=7.0           # 魚過濾低吃分數門檻
INST_FISH_DAYS=90             # 魚過濾回溯天數
INST_TOP_N=150                # 從台股市值前 150 大尋找標的
```
