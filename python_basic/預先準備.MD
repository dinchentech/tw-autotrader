# 事前準備 — 工具安裝與環境設定

> 📖 **本課程所有 .md 檔案都用 VS Code 的「預覽模式」閱讀**
> VS Code 有兩種顯示模式：
> - **編輯模式**（Edit）：顯示原始碼，包含 `##`、`**粗體**`、`|` 表格線等標記符號 — 這是寫文章用的
> - **預覽模式**（Preview）：顯示渲染後的結果，標題變大字、表格有框線、**粗體**變粗體 — **這是你讀文章該用的模式**
>
> **請習慣用預覽模式閱讀，不要被原始碼嚇到。**

### 一、安裝 VS Code（看 MD 檔案用）

#### 1.1 下載與安裝
1. 打開瀏覽器，到 https://code.visualstudio.com/
2. 點藍色「Download for Windows」按鈕
3. 下載後執行安裝程式，全部按「下一步」就好
4. 安裝完成後，打開 VS Code

#### 1.2 設定字型大小（18 號）
1. 打開 VS Code
2. 左下角有一個齒輪圖示 ⚙️ → 點「Settings」（設定）
3. 在搜尋框輸入 `font size`
4. 找到「Editor: Font Size」，把 14 改成 **18**
5. 關閉設定視窗，字型就會變大

#### 1.2b 預覽時如何放大/縮小字型

預覽模式下的字型縮放方式和一般瀏覽器一樣：

| 動作 | 快捷鍵 |
|------|--------|
| **放大** | `Ctrl + +`（按住 Ctrl 再按加號） |
| **縮小** | `Ctrl + -`（按住 Ctrl 再按減號） |
| **重設 100%** | `Ctrl + 0`（數字零） |

> 編輯模式（原始碼）的字型在第 1.2 節設定過。預覽模式下用 `Ctrl + +/-` 縮放是獨立的，不影響編輯區字型。
> VS Code 會記住你最後的縮放比例，下次開啟時自動沿用。

#### 1.3 用 VS Code 打開 MD 檔案（用預覽模式閱讀）

1. 直接把 `.md` 檔案**拖進** VS Code 視窗
2. 你會看到原始碼（編輯模式）— 這是正常的，不用慌
3. 按 **`Ctrl + Shift + V`** 切換到預覽模式 ← 這是你主要用的模式
4. 也可以按 **`Ctrl + K` 再按 `V`** 開啟側邊預覽（左邊原始碼、右邊渲染結果）

**重點**：你只需要記住 `Ctrl + Shift + V` 這組快捷鍵，按下去就能看到美美的文章了。

### 二、安裝 PyCharm（寫 Python 用, wsl備份檔內已經有裝好）

#### 2.1 下載與安裝
1. 打開瀏覽器，到 https://www.jetbrains.com/pycharm/download/
2. 選擇 **Community Edition（社群版）** — 免費，功能就夠了
3. 下載後執行安裝程式
   - 安裝過程中，勾選：
     - 「Create Desktop Shortcut」
     - 「Add 'Open Folder as Project'」
     - 「Add launchers dir to the PATH」
4. 安裝完成後，開啟 PyCharm

#### 2.2 打開本專案（Open Project）
1. 啟動 PyCharm，會看到 Welcome 視窗
2. 點 **「Open」**（開啟）
3. 瀏覽到本專案的資料夾（例如 `C:\Users\你的名字\Documents\github\tw-autotrader`）
4. 選取該資料夾，點「OK」
5. PyCharm 會花一點時間載入專案（第一次會建立索引）

#### 2.3 設定 Python 直譯器為 .venv

設定 Python 直譯器（Interpreter）讓 PyCharm 知道用哪個 Python 執行你的程式：

1. 打開 PyCharm → 上方選單 **File → Settings**（或按 `Ctrl+Alt+S`）
2. 左邊找到 **Project: tw-autotrader → Python Interpreter**
3. 右邊點齒輪 ⚙️ → **Add Local Interpreter**
4. 選 **Existing**（使用現有的）
5. 在 Interpreter 欄位右邊的資料夾圖示，瀏覽到：
   ```
   C:\Users\你的名字\Documents\github\tw-autotrader\.venv\Scripts\python.exe
   ```
6. 如果找不到 `.venv`，點 Add Local Interpreter → **New**（建立新的）
   - 選 Virtualenv
   - Location 填：專案資料夾 `\.venv`
   - Base interpreter 選你安裝的 Python（如 `C:\Python310\python.exe`）
   - 按 OK 建立
7. 建立好後，PyCharm 會自動安裝套件（可能需要幾分鐘）
8. 確認右下角顯示的是 `.venv` 的 Python 版本

#### 2.4 如何除錯（Debug）— 設斷點、觀察變數、開終端機

**設斷點（Breakpoint）：**
1. 在 PyCharm 打開一個 `.py` 檔案（例如 `week-01-calc_return.py`）
2. 在你要暫停的那一行，**左邊的灰色區域點一下** → 會出現一個紅色圓點 ⏺
3. 紅色圓點就是斷點。當程式執行到這一行時，它會暫停讓你看變數

**執行除錯：**
1. 在要 debug 的檔案上按右鍵
2. 選 **Debug '檔案名稱'**（或按 `Shift+F9`）
3. 程式會開始執行，直到碰到斷點為止暫停

**觀察變數值：**
1. 程式在斷點暫停時，PyCharm 下方會出現 Debug 視窗
2. 在 **Variables** 分頁可以看到所有變數的當前值
3. 也可以直接把你的滑鼠移到程式碼的變數上（例如 `return_rate`），會跳出提示框顯示它的值
4. 要繼續執行：按 F8（逐行執行）或 F9（跑到下一個斷點）

**開終端機（Terminal）：**
1. PyCharm 下方有一個 **Terminal** 分頁標籤
2. 如果沒看到，按選單 **View → Tool Windows → Terminal**（或 `Alt+F12`）
3. 在 Terminal 中你可以直接輸入 Python 指令：
   ```bash
   python week-01-calc_return.py
   ```
   （PyCharm 會自動使用你剛剛設定的 .venv Python）
4. 注意：如果你在 WSL 環境執行，需要在 Terminal 中先切到 WSL（見第三章）

**常用除錯快捷鍵總表：**

| 按鍵 | 功能 |
|------|------|
| `Shift+F9` | Debug 執行（從目前的檔案啟動） |
| `F8` | 跳過這一行（Step Over） |
| `F7` | 跳進函式（Step Into） |
| `Shift+F8` | 跳出函式（Step Out） |
| `F9` | 繼續執行到下一的斷點（Resume） |
| `Ctrl+F2` | 停止除錯 |

### 三、安裝 WSL2（Linux 子系統）

#### 3.1 WSL2 是什麼
WSL = Windows Subsystem for Linux。簡單說，就是在 Windows 裡面開一個「Linux 視窗」，你可以直接在裡面打 Linux 指令、跑 Python 程式，而且效能很好。

#### 3.2 安裝 WSL2

**方法一：一條指令安裝（最快，Windows 10 版本 2004+ 或 Windows 11）**
1. 按 `Win + R`，輸入 `powershell`，按 `Ctrl+Shift+Enter`（以系統管理員身分執行）
2. 在 PowerShell 中輸入：`wsl --install`
3. 等待它自動下載並安裝。完成後**重新開機**。
4. 重新開機後，系統會自動啟動 Ubuntu 設定畫面，請輸入：
   - 使用者名稱：例如 `frank`
   - 密碼：輸入兩次（輸入時不會顯示字元，這是正常的）
5. 安裝完成！你會看到 `frank@電腦名稱:~$`

**方法二：手動安裝（如果方法一失敗）**
1. 按 `Win + R`，輸入 `appwiz.cpl`，按 Enter
2. 左邊點「開啟或關閉 Windows 功能」
3. 勾選：「適用於 Linux 的 Windows 子系統」和「虛擬機器平台」
4. 按確定，重開機
5. 重開機後，到 Microsoft Store 搜尋 **Ubuntu**（選擇最新版），按「安裝」
6. 安裝後啟動 Ubuntu，輸入使用者名稱和密碼

#### 3.3 確認 WSL 版本為 WSL2
```powershell
wsl --list --verbose
```
確認 VERSION 是 2。如果是 1，執行 `wsl --set-version Ubuntu 2`

#### 3.4 啟動 WSL
- 直接搜尋並打開 **Ubuntu** 應用程式
- 或在 PowerShell 輸入 `wsl` 按 Enter
看到 `frank@電腦名稱:~$` 就代表你已經在 Linux 環境中了。

### 四、匯入 WSL 備份（Import）

#### 4.1 什麼時候需要 Import
- 別人（或你自己）把設定好的 WSL 環境打包給你
- 換電腦時要還原開發環境
- 團隊統一發佈標準開發環境

#### 4.2 匯入 .tar 備份檔
假設你有一份備份檔 `tw-autotrader-wsl.tar` 放在 `D:\backup\`：
1. 打開 **PowerShell**（以系統管理員身分執行）
2. `mkdir D:\wsl\Ubuntu-Backup`
3. `wsl --import Ubuntu-Backup D:\wsl\Ubuntu-Backup D:\backup\tw-autotrader-wsl.tar`
   - Ubuntu-Backup → 你幫這個 WSL 發行版取的名字
   - D:\wsl\Ubuntu-Backup → 存放路徑
   - D:\backup\tw-autotrader-wsl.tar → 備份檔的路徑
4. 設定預設使用者：`wsl -d Ubuntu-Backup --user frank`
5. 啟動：`wsl -d Ubuntu-Backup`

#### 4.3 匯入 .vhdx 磁碟檔
```powershell
wsl --import-in-place Ubuntu-Backup D:\backup\tw-autotrader.vhdx
```

#### 4.4 管理已安裝的 WSL 發行版
```powershell
# 查看所有
wsl --list --verbose

# 設為預設
wsl --set-default Ubuntu-Backup

# 登出（關閉，保留檔案）
wsl --terminate Ubuntu-Backup

# 刪除（小心，清掉所有檔案）
wsl --unregister Ubuntu-Backup
```

#### 4.5 匯出你目前的 WSL（做備份）
```powershell
wsl --shutdown
wsl --export Ubuntu D:\backup\tw-autotrader-wsl-20240606.tar
```

### 五、Windows 目錄與 WSL 目錄對應關係

如果你有用 WSL（Windows Subsystem for Linux，就是本專案在開發環境中會用到的 Linux 子系統），要知道檔案在兩個系統之間的對應方式。

#### 5.1 WSL 是什麼
WSL 讓你在 Windows 裡面跑一個 Linux 系統。本專案的一些指令（如 `python backtest.py`）可能在 WSL 底下執行。

#### 5.2 Windows 路徑 → WSL 路徑

當你在 WSL 終端機中時，要找到 Windows 的檔案，用這個規則：

| Windows 路徑 | WSL 對應路徑 |
|-------------|-------------|
| `C:\Users\你的名字\Documents\...` | `/mnt/c/Users/你的名字/Documents/...` |
| `D:\projects\...` | `/mnt/d/projects/...` |
| `C:\` 整顆硬碟 | `/mnt/c/` |

**舉例：**

```
Windows 路徑：
C:\Users\frank\Documents\github\tw-autotrader

在 WSL 中要寫成：
/mnt/c/Users/frank/Documents/github/tw-autotrader
```

#### 5.3 WSL 路徑 → Windows 路徑

反過來，如果你在 Windows（如 PyCharm、檔案總管）中要找到 WSL 的檔案：

```
WSL 路徑：
/home/frank/projects/tw-autotrader

在 Windows 檔案總管要打：
\\wsl$\Ubuntu\home\frank\projects\tw-autotrader
```

或在檔案總管的網址列直接輸入 `\\wsl$\` 可以看到所有安裝的 WSL 發行版。

#### 5.4 開發流程建議

因為本專案可以在 Windows（原生 Python）或 WSL（Linux Python）中執行，建議如下：

- **閱讀 MD 檔案** → 用 VS Code（Windows）
- **寫程式、除錯** → 用 PyCharm（Windows）
- **跑 backtest.py、執行指令** → 用 PyCharm 的 Terminal（Windows）或 WSL 終端機

如果你不確定要選哪個，先用 **Windows + PyCharm** 就好，這是對初學者最友善的組合。
